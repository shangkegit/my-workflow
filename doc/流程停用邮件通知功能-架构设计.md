# 流程停用邮件通知功能 - 架构设计文档

## 一、需求分析

### 1.1 功能概述

在流程部署管理中，当管理员执行**流程停用（挂起）**操作时，系统自动发送邮件通知相关用户。

### 1.2 功能点拆解

#### 1.2.1 触发时机
- **入口**：`FlowController.suspendProcessDefinition()` 接口
- **操作**：管理员在"部署管理"页面点击"停用"按钮
- **参数**：流程定义ID (pdid)

#### 1.2.2 通知对象及内容

| 通知对象 | 通知内容 | 邮箱来源 |
|---------|---------|---------|
| **管理员** | 本次停用的流程信息摘要，包括：<br>- 流程名称<br>- 流程版本<br>- 停用时间<br>- 受影响的在途实例数量 | 从系统配置读取 |
| **申请人** | 自己发起的被停用流程实例信息：<br>- 流程名称<br>- 发起时间<br>- 当前节点<br>- 流程状态 | 从 `sys_user` 表的 `email` 字段读取 |

#### 1.2.3 业务逻辑

1. **查询被停用流程的所有在途实例**
   - 使用 `RuntimeService.createProcessInstanceQuery()`
   - 过滤条件：`processDefinitionId = pdid`
   - 排除已挂起的实例（可选）

2. **按申请人分组**
   - 使用 `ProcessInstance.getStartUserId()` 获取申请人
   - Map<申请人, List<流程实例>> 分组

3. **分别发送邮件**
   - 管理员邮件：汇总所有受影响的流程实例
   - 申请人邮件：仅包含自己发起的流程实例

### 1.3 非功能性需求

- **异步发送**：邮件发送不应阻塞主流程
- **容错处理**：邮件发送失败不应影响流程停用操作
- **可配置**：邮件开关、管理员邮箱等应支持配置
- **日志记录**：记录邮件发送状态，便于排查

### 3.2 核心代码示例

#### 3.2.1 MailService实现

```java
@Service
public class MailServiceImpl implements MailService {
    
    @Autowired
    private JavaMailSender mailSender;
    
    @Value("${mail.enabled:false}")
    private boolean mailEnabled;
    
    @Value("${mail.admin-email}")
    private String adminEmail;
    
    @Value("${mail.from}")
    private String from;
    
    @Autowired
    private TaskExecutor taskExecutor; // 异步线程池
    
    @Override
    public void sendSuspendNoticeToAdmin(ProcessDefinition pd, List<ProcessInstance> instances) {
        if (!mailEnabled || StringUtils.isEmpty(adminEmail)) {
            return;
        }
        
        String subject = "【流程停用通知】" + pd.getName();
        String content = buildAdminMailContent(pd, instances);
        sendMailAsync(adminEmail, subject, content);
    }
    
    @Override
    public void sendSuspendNoticeToApplicant(String userName, String userEmail, List<ProcessInstance> instances) {
        if (!mailEnabled || StringUtils.isEmpty(userEmail)) {
            return;
        }
        
        String subject = "【您的流程已被停用】";
        String content = buildApplicantMailContent(userName, instances);
        sendMailAsync(userEmail, subject, content);
    }
    
    @Override
    @Async
    public void sendMailAsync(String to, String subject, String content) {
        try {
            MimeMessage message = mailSender.createMimeMessage();
            MimeMessageHelper helper = new MimeMessageHelper(message, true, "UTF-8");
            helper.setFrom(from);
            helper.setTo(to);
            helper.setSubject(subject);
            helper.setText(content, true); // true表示HTML格式
            mailSender.send(message);
            log.info("邮件发送成功: to={}, subject={}", to, subject);
        } catch (Exception e) {
            log.error("邮件发送失败: to={}, subject={}, error={}", to, subject, e.getMessage());
        }
    }
    
    private String buildAdminMailContent(ProcessDefinition pd, List<ProcessInstance> instances) {
        StringBuilder sb = new StringBuilder();
        sb.append("<h2>流程停用通知</h2>");
        sb.append("<p>流程名称：").append(pd.getName()).append("</p>");
        sb.append("<p>流程版本：").append(pd.getVersion()).append("</p>");
        sb.append("<p>停用时间：").append(new SimpleDateFormat("yyyy-MM-dd HH:mm:ss").format(new Date())).append("</p>");
        sb.append("<p>受影响的在途实例数量：").append(instances.size()).append("</p>");
        
        if (!instances.isEmpty()) {
            sb.append("<h3>受影响的流程实例：</h3>");
            sb.append("<table border='1'>");
            sb.append("<tr><th>实例ID</th><th>发起人</th><th>发起时间</th></tr>");
            for (ProcessInstance pi : instances) {
                sb.append("<tr>");
                sb.append("<td>").append(pi.getId()).append("</td>");
                sb.append("<td>").append(pi.getStartUserId()).append("</td>");
                sb.append("<td>").append(pi.getStartTime()).append("</td>");
                sb.append("</tr>");
            }
            sb.append("</table>");
        }
        return sb.toString();
    }
    
    private String buildApplicantMailContent(String userName, List<ProcessInstance> instances) {
        StringBuilder sb = new StringBuilder();
        sb.append("<h2>流程停用通知</h2>");
        sb.append("<p>").append(userName).append(" 您好，</p>");
        sb.append("<p>您发起的以下流程已被管理员停用：</p>");
        
        sb.append("<table border='1'>");
        sb.append("<tr><th>流程名称</th><th>发起时间</th></tr>");
        for (ProcessInstance pi : instances) {
            sb.append("<tr>");
            sb.append("<td>").append(pi.getProcessDefinitionName()).append("</td>");
            sb.append("<td>").append(pi.getStartTime()).append("</td>");
            sb.append("</tr>");
        }
        sb.append("</table>");
        
        sb.append("<p>如有疑问，请联系管理员。</p>");
        return sb.toString();
    }
}
```

#### 3.2.2 FlowController修改

```java
@ApiOperation("挂起一个流程定义")
@RequestMapping(value = "/suspendProcessDefinition", method = RequestMethod.GET)
@ResponseBody
public AjaxResult suspendProcessDefinition(@RequestParam("pdid") String pdid, @RequestParam("flag") Boolean flag,
                                           @RequestParam(value="date", required = false) String date) throws Exception {
    // 1. 查询流程定义信息
    ProcessDefinition pd = repositoryService.createProcessDefinitionQuery()
            .processDefinitionId(pdid)
            .singleResult();
    
    if (pd == null) {
        return AjaxResult.error("流程定义不存在");
    }
    
    // 2. 查询受影响的在途流程实例（在挂起之前查询）
    List<ProcessInstance> instances = runtimeService.createProcessInstanceQuery()
            .processDefinitionId(pdid)
            .active() // 只查询活动状态的实例
            .list();
    
    // 3. 执行挂起操作
    if (StringUtils.isNotEmpty(date)) {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        repositoryService.suspendProcessDefinitionById(pdid, flag, sdf.parse(date));
    } else {
        repositoryService.suspendProcessDefinitionById(pdid, flag, null);
    }
    
    // 4. 发送邮件通知（异步）
    if (instances.size() > 0) {
        sendSuspendNotifications(pd, instances);
    }
    
    return AjaxResult.success();
}

/**
 * 发送流程停用通知
 */
private void sendSuspendNotifications(ProcessDefinition pd, List<ProcessInstance> instances) {
    try {
        // 1. 发送管理员通知
        mailService.sendSuspendNoticeToAdmin(pd, instances);
        
        // 2. 按申请人分组
        Map<String, List<ProcessInstance>> groupByApplicant = instances.stream()
                .collect(Collectors.groupingBy(ProcessInstance::getStartUserId));
        
        // 3. 发送申请人通知
        for (Map.Entry<String, List<ProcessInstance>> entry : groupByApplicant.entrySet()) {
            String userName = entry.getKey();
            List<ProcessInstance> userInstances = entry.getValue();
            
            // 查询用户邮箱
            SysUser user = userService.selectUserByUserName(userName);
            if (user != null && StringUtils.isNotEmpty(user.getEmail())) {
                mailService.sendSuspendNoticeToApplicant(
                    user.getNickName(), 
                    user.getEmail(), 
                    userInstances
                );
            }
        }
    } catch (Exception e) {
        // 邮件发送失败不影响主流程
        log.error("发送流程停用通知失败: {}", e.getMessage(), e);
    }
}
```

## 五、风险点与解决方案

### 5.1 技术风险

| 风险点 | 影响 | 解决方案 |
|-------|------|---------|
| **SMTP服务器不稳定** | 邮件发送失败 | 1. 异步发送，不阻塞主流程<br>2. 记录失败日志，支持手动重发<br>3. 考虑引入消息队列 |
| **大量邮件发送慢** | 影响用户体验 | 1. 使用线程池异步发送<br>2. 考虑批量发送优化<br>3. 添加发送限流 |
| **邮箱地址无效** | 邮件退回 | 1. 发送前校验邮箱格式<br>2. 记录退信日志 |

### 5.2 业务风险

| 风险点 | 影响 | 解决方案 |
|-------|------|---------|
| **申请人无邮箱** | 无法通知 | 1. 在用户管理中强制要求填写邮箱<br>2. 通知失败记录日志 |
| **邮件开关误关** | 无法发送通知 | 1. 配置文件注释说明<br>2. 启动时检查配置 |
| **敏感信息泄露** | 安全问题 | 1. 邮件内容不包含敏感业务数据<br>2. 仅通知流程名称、时间等基本信息 |

### 5.3 运维风险

| 风险点 | 影响 | 解决方案 |
|-------|------|---------|
| **SMTP密码泄露** | 邮箱被盗用 | 1. 使用授权码而非密码<br>2. 配置文件加密（Jasypt） |
| **邮件被拦截** | 进垃圾箱 | 1. 配置SPF、DKIM记录<br>2. 使用企业邮箱服务 |
